# Plan: Card Fan Carousel → Flashcard UI

## Context

The user wants to integrate the card-fan-carousel design pattern into the flashcard viewer. The React component uses GSAP for animation, but this project is zero-dependency static HTML/CSS/JS.

**Data**: 116 flashcards (104 N5, 8 N4, 4 N3). Each has `front` (Japanese word), `back` (reading, meaning, POS, JLPT level, occurrence count), and YouTube timestamp links.

## Design Approach

### Visual Concept
Cards spread in a fan shape. The center card is the "active" card that can be flipped by clicking. Clicking off-center cards or arrow buttons cycles the deck. Pagination dots show position. This replaces the current single-card + prev/next nav.

### Fan Layout (7 visible slots at a time)
```
     card-0 card-1 card-2  [CENTER] card-4 card-5 card-6
       -21°   -14°   -7°      0°     +7°   +14°   +21°
```

Fan positions modeled after the React component but adapted for our card sizes (400px wide, ~300px tall).

### Why 7 Max Visible
The React component uses `MAX_VISIBLE = 7`. For 116 cards, users paginate through. This creates a manageable fan that doesn't overflow.

## Implementation Plan

### 1. Rewrite `ui/flashcards.css` — Add fan layout CSS

Keep existing `.flip-card-inner`, `.flip-card-face` etc. Add:
- `.fan-deck` — perspective container (replaces `.deck`)
- `.fan-track` — flexbox container for fan cards
- Fan card wrapper `.fan-card-slot` — positioned absolutely, sized to card dimensions
- 7 `.fan-pos-N` classes with `transform: rotate($deg) translateX($x) translateY($y) scale($s)` and `z-index`
- Transitions on all transform properties: `transition: all 0.5s cubic-bezier(0.4, 0.2, 0.2, 1)`
- `.fan-card-slot.center` highlight (subtle glow/border accent)
- `.fan-card-slot` hover: lift slightly
- Pagination dots at bottom (styled like existing nav, JLPT color dots for active)

### 2. Rewrite `ui/app.js` — Fan carousel logic

Replace the single-card render with:
- `renderFan()` — calculates visible window (7 cards centered on `currentIndex`), creates/updates DOM slots
- Each slot contains a flip-card-inner with flip-card-face front/back
- Click on center card → flip (existing flip logic)
- Click on left card → cycle left (decrement centerIndex)
- Click on right card → cycle right (increment centerIndex)
- Arrow buttons → cycle with animation
- Keyboard: left/right → cycle, Space/Enter → flip center card

Data flow:
1. `currentIndex` = center card index
2. `visibleWindow(centerIndex)` = 7 indices (wrapping for edges)
3. For each visible card: populate front/back from `cards[cardIndex]`
4. Position each card in its fan slot with CSS class

### 3. Rewrite `ui/index.html` — Fan deck markup

Replace:
```html
<div class="deck" id="deck">
  <div class="flip-card-inner" id="flipCardInner">...</div>
</div>
```

With:
```html
<div class="fan-deck" id="fanDeck">
  <div class="fan-track" id="fanTrack">
    <!-- 7 fan-card slots generated by JS -->
  </div>
</div>
```

Keep header, breakdown, controls, nav-ui, states, and script tag.

### 4. Fan position CSS — responsive scaling

Adapt the React component's `getResponsiveMultiplier()`:
- Desktop (>=1024px): 1.0x (full fan spread)
- Tablet (768-1023px): 0.75x (tighter fan)
- Mobile-small (640-767px): 0.5x
- Tiny (<480px): 0.35x

Use CSS custom properties + calc() where possible, JS for dynamic scaling on resize.

### 5. Styling — Design language continuity

| Element | Style |
|---------|-------|
| Fan deck background | `#1a1a2e` (body bg) — no separate container |
| Card faces | Existing `.flip-card-face` gradient + border |
| Center card accent | `box-shadow: 0 0 30px rgba(136, 136, 204, 0.2)` |
| Pagination dots | Active: JLPT color of center card; Inactive: `#3a3a60` |
| Arrow buttons | Existing `.nav-btn` style |
| Entry animation | Cards stagger in from bottom with opacity 0→1 (CSS animation, no GSAP) |
